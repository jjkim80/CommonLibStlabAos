주석 작성 규칙

클래스 설명 주석
	Import 다음에 기술
	블록주석을 사용
	각 라인은 *로 시작
	해당 클래스에 대한 기술과 용도 기술

/**
* 해당 클래스에 대한 기능과 용도를 적을 것.
*
* @author 최종 수정자
* @version 1.0 작업 내용
* @date 작업일자(yyyy-MM-dd)
*/

멤버함수 설명 주석
	함수의 상단에 위치
	블록주석을 사용
	메소드 기능 설명은 한, 두줄로 간략하게
	메서드의 파라미터를 type명과 변수명을 적고 간략하게 설명
	리턴 변수, 예외사항 설명



/**
* 메소드의 기능을 설명. 간략하게 적을 것
*
* @ param int a 메서드의 파라미터 설명 (파라미터가 없을 경우 생략 가능)
* @ return 반환하는 것 설명 (반환 값이 없을 경우 생략 가능)
* @ exception 예외사항 (예외 사항이 없을 경우 생략 가능)
*/

네이밍 규칙

클래스 이름
클래스 이름은 UpperCamelCase로 작성한다.
클래스 이름은 일반적으로 명사 또는 명사구로 작성한다. 
ex) Charater, HashTest

메소드 이름
메소드 이름은 lowerCamelCase로 작성한다.
메소드 이름은 일반적으로 동사 또는 동사구로 작성한다. 
ex) sendMessage, stop

상수 이름
상수 이름은 CONSTANT_CASE를 사용. 각 단어는 단일 밑줄로 다음 단어와 구분 
ex) COMMA_JOINER


매개 변수, 지역변수 이름
매개 변수, 지역변수 이름은 lowerCamelCase로 작성한다.
일반적으로 명사 또는 명사구를 사용한다.
ex) computedValue, index
